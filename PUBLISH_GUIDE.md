# SDS Motion — NPM Publishing Guide

## Step-by-Step: Publish to NPM

---

### 1. Folder Structure (before publishing)

```
sds-motion/
├── src/
│   └── motion.css          ← paste all @keyframes + class definitions here
├── dist/
│   ├── motion.css          ← generated by build
│   └── motion.min.css      ← generated by build (minified)
├── package.json
├── README.md
├── LICENSE
└── .npmignore
```

---

### 2. Install Build Dependencies

```bash
npm install --save-dev postcss postcss-cli autoprefixer clean-css-cli
```

---

### 3. Create `postcss.config.js`

```js
module.exports = {
  plugins: [
    require('autoprefixer')
  ]
};
```

---

### 4. Place Your CSS

Copy all animation `@keyframes` and class definitions into `src/motion.css`.

---

### 5. Build

```bash
npm run build
```

This generates:
- `dist/motion.css` — full, readable
- `dist/motion.min.css` — minified for production

---

### 6. Create NPM Account

Go to https://www.npmjs.com and register, then:

```bash
npm login
# Enter: username, password, email, OTP (if 2FA enabled)
```

---

### 7. Verify Package Name Availability

```bash
npm search sds-motion
```

---

### 8. Dry Run (check what gets published)

```bash
npm pack --dry-run
```

---

### 9. Publish

```bash
npm publish --access public
```

---

### 10. Verify on NPM

Visit: https://www.npmjs.com/package/sds-motion

CDN is auto-live at:
- https://cdn.jsdelivr.net/npm/sds-motion/dist/motion.min.css
- https://unpkg.com/sds-motion/dist/motion.min.css

---

### 11. Update Version (for future releases)

```bash
npm version patch   # 1.0.0 → 1.0.1  (bug fixes)
npm version minor   # 1.0.0 → 1.1.0  (new animations)
npm version major   # 1.0.0 → 2.0.0  (breaking changes)
npm publish --access public
```

---
